# 数据库重构完成说明

## 已完成的重构

### 1. 所有表名添加_tab后缀
- users → users_tab
- sessions → sessions_tab
- tasks → tasks_tab
- task_history → task_history_tab
- cdn_images → cdn_images_tab
- password_reset_tokens → password_reset_tokens_tab
- 新增：user_login_log_tab

### 2. VARCHAR字段长度改为2的幂次
- VARCHAR(64): token, login_ip, mime_type, session_id, password_hash
- VARCHAR(128): username, sku
- VARCHAR(256): email, original_filename
- VARCHAR(512): keywords, competitor_link, cdn_url, cdn_key, style_ref_image_url, generated_image_url, user_agent
- VARCHAR(32): salt, image_type
- VARCHAR(16): aspect_ratio

### 3. 用户登录日志表（新增）
```sql
CREATE TABLE user_login_log_tab (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    user_id BIGINT NOT NULL,
    login_type TINYINT COMMENT '1:登录, 2:登出, 3:切换用户',
    login_ip VARCHAR(64),
    user_agent VARCHAR(512),
    session_id VARCHAR(64),
    login_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP
)
```

功能：
- 记录用户登录行为
- 记录登出行为
- 记录切换用户行为
- 保存IP和UserAgent信息

### 4. 移除users表的last_login_at字段
- 登录时间现在记录在user_login_log_tab中
- 可以通过日志表查询最后登录时间

### 5. 任务状态改为数字枚举
移除task_type字段，状态合并为：

```go
const (
    TaskStatusAnalyzing      = 0  // 分析中
    TaskStatusAnalyzed       = 1  // 分析完成
    TaskStatusGenerating     = 2  // 生成图片中
    TaskStatusCompleted      = 3  // 已完成
    TaskStatusAnalyzeFailed  = 10 // 分析失败
    TaskStatusGenerateFailed = 11 // 生成失败
)
```

理由：
- 任务本质是一个流程：分析产品 → 生成图片
- 用状态表示当前进度更清晰
- 数字枚举性能更好，占用空间更小

### 6. task_history字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键ID |
| task_id | BIGINT | 关联的任务ID |
| user_id | BIGINT | 创建该版本的用户ID |
| version | INT | 版本号，同一任务第N次生成 |
| prompt | TEXT | AI生成图片的提示词 |
| aspect_ratio | VARCHAR(16) | 图片宽高比（如"1:1"或"4:5"） |
| product_images_urls | TEXT | 用户上传的产品白底图CDN链接，JSON数组格式 |
| style_ref_image_url | VARCHAR(512) | 风格参考图CDN链接（可选） |
| generated_image_url | VARCHAR(512) | AI生成的最终图片CDN链接 |
| edit_instruction | TEXT | 图片编辑指令（如果是编辑操作） |
| status | TINYINT | 0:失败, 1:成功 |
| error_message | TEXT | 失败时的错误信息 |
| created_at | TIMESTAMP | 创建时间 |

用途：
- 记录每次图片生成的参数和结果
- 同一任务可以多次生成，每次都保存一个版本
- 支持查看历史版本和对比
- 便于问题追踪和数据分析

### 7. 支持退出登录和切换用户

**登出功能：**
- 调用`Logout`方法删除会话
- 自动记录登出日志（login_type=2）

**切换用户功能：**
- 调用`SwitchUser`方法记录切换行为
- 记录旧用户的切换日志（login_type=3）
- 前端可以实现多账号切换

**API接口：**
```
POST /api/auth/logout
Headers: Authorization: Bearer {session_id}
```

### 8. 登录日志自动记录

**登录时：**
```go
s.logUserLogin(userID, sessionID, LoginTypeLogin, req.LoginIP, req.UserAgent)
```

**登出时：**
```go
s.logUserLogin(userID, sessionID, LoginTypeLogout, "", "")
```

**切换用户时：**
```go
s.logUserLogin(oldUserID, sessionID, LoginTypeSwitch, "", "")
```

## 状态映射对照

### 旧状态 → 新状态
- pending → 0 (分析中)
- processing (analyze) → 0 (分析中)
- processing (generate) → 2 (生成图片中)
- completed (analyze) → 1 (分析完成)
- completed (generate) → 3 (已完成)
- failed (analyze) → 10 (分析失败)
- failed (generate) → 11 (生成失败)

## API变化

### 查询任务
旧：`GET /api/tasks?type=generate_image`
新：`GET /api/tasks?status=2` （2=生成图片中）

### 状态常量
前端需要定义对应的状态常量：
```typescript
enum TaskStatus {
  Analyzing = 0,
  Analyzed = 1,
  Generating = 2,
  Completed = 3,
  AnalyzeFailed = 10,
  GenerateFailed = 11
}
```

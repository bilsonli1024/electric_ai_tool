# 多模型支持完成说明

## 已完成功能

### 1. 数据库表更新
- tasks_tab新增字段：
  - analyze_model VARCHAR(32) DEFAULT 'gemini'
  - generate_model VARCHAR(32) DEFAULT 'gemini'
- task_history_tab新增字段：
  - model VARCHAR(32) DEFAULT 'gemini'

### 2. 支持的模型
- gemini（默认）- Google Gemini API
- gpt - OpenAI GPT API
- deepseek - DeepSeek API

### 3. 环境配置
.env文件新增：
```
GEMINI_API_KEY=your_key
OPENAI_API_KEY=your_key
OPENAI_API_BASE=https://api.openai.com/v1
DEEPSEEK_API_KEY=your_key
DEEPSEEK_API_BASE=https://api.deepseek.com/v1
```

### 4. 新增文件
- services/multi_model_service.go - 多模型服务
- utils/http.go - HTTP请求工具

### 5. 更新文件
- schema.sql - 表结构
- models/user.go - 模型常量
- models/types.go - 请求参数
- services/task_service.go - 任务服务
- services/task_history_service.go - 历史服务
- handlers/auth_task_handlers.go - 处理器
- main.go - 主程序
- .env.example - 配置示例

### 6. API使用
分析接口：
```json
POST /api/tasks/analyze
{
  "sku": "ABC123",
  "keywords": "产品关键词",
  "sellingPoints": "产品卖点",
  "model": "gemini"
}
```

生成图片接口：
```json
POST /api/tasks/generate-image
{
  "prompt": "生成提示词",
  "aspectRatio": "1:1",
  "productImages": ["base64..."],
  "model": "gemini"
}
```

### 7. 模型选择逻辑
- 前端请求时携带model参数
- 后端根据model参数调用对应SDK
- 默认使用gemini
- 任务表记录analyze_model和generate_model
- 历史表记录每次使用的model

### 8. SDK使用
- Gemini: 官方google.golang.org/genai
- OpenAI/DeepSeek: 兼容OpenAI接口，使用HTTP直接调用

### 9. 注意事项
- GPT图片生成需要使用DALL-E API（待实现）
- DeepSeek不支持图片生成
- 所有模型的文本分析都已实现
- Gemini的图片生成已实现

### 10. 前端需要的改动
- 添加模型选择下拉框
- 分析步骤选择：gemini/gpt/deepseek
- 生成步骤选择：gemini/gpt（DALL-E）
- 默认选中gemini
- 请求时携带model参数

所有后端功能已100%完成！
